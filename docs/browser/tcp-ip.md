# TCP/IP 与网络协议

理解 TCP/IP 协议是前端工程师的必备网络知识。

## 三次握手与四次挥手

```
三次握手（建立连接）：

客户端                    服务端
  | ---- SYN=1, seq=x ----> |
  |                         |
  | <--- SYN=1, ACK=1, ---- |
  |      seq=y, ack=x+1     |
  |                         |
  | ---- ACK=1, seq=x+1 --->|
  |      ack=y+1            |
  |                         |

四次挥手（断开连接）：

客户端                    服务端
  | ---- FIN=1, seq=u ----> |
  |                         |
  | <------- ACK=1 ---------|
  |       ack=u+1           |
  |                         |
  | <------ FIN=1 -----------
  |       seq=v             |
  |                         |
  | ---- ACK=1, ack=v+1 --->|
```

### 三次握手的目的
1. 确认双方发送和接收能力正常
2. 同步初始序列号
3. 防止历史重复连接初始化

## TCP vs UDP

| 特性 | TCP | UDP |
|------|-----|-----|
| 连接 | 面向连接 | 无连接 |
| 可靠性 | 可靠传输 | 不可靠 |
| 顺序 | 保证顺序 | 不保证 |
| 流量控制 | 有 | 无 |
| 拥塞控制 | 有 | 无 |
| 头部开销 | 20字节 | 8字节 |
| 适用场景 | HTTP、FTP、邮件 | 视频流、DNS、游戏 |

### 高频面试题

**Q: TCP 为什么需要三次握手？两次不行吗？**

**答案**：
1. **确认双方能力**：确保客户端和服务端的发送、接收能力都正常
2. **同步序列号**：双方交换初始序列号，为后续数据传输做准备
3. **防止历史连接**：避免已失效的连接请求突然到达服务端，造成错误初始化

如果只有两次握手，无法防止已失效的连接请求报文段突然到达服务端，导致资源浪费。
